# 竞品分析 insight 1

## 统一工作面

•刚才我们提到的两个维度是关于“统一工作面”的。  
•统一工作面的目的：提升跨层级协作中，信息传递的“保真度”和“效率”  
•需求层（意图）  
•决策层（假设、约束、因果）  
•实现层（代码、资源）  
•现在市面上的工具多数都停留在其中一个层级。  
•分类的意义在于：先判断竞品“所在的生态位置”，再理解它在该层级内的得失、以及对我们的启示。  

## 分层级选择竞品（组一）

注意  
我们的竞品不是一个单一的产品  

### 需求层 · 意图

意图生成  
我要达成什么目标？为什么现在要做？  

### 决策层 · 假设、约束、因果

意图澄清与封版  
假设是什么？约束是什么？什么情况下算“成功 / 失败”？  

### 实现层 · 代码、资源

任务化与分发  
意图如何被拆解、被分配、被执行？  

实现与变更  
代码 / 设计 / 资源如何演化？为什么会变？  

复盘与资产化  
哪些假设被证实 / 证伪？下次还能不能直接用？  

结果回流  
数据、结果是否回到最初的假设？

## 共性问题

我们决定了，但系统没记住。  
多数决策都是一次性决策，没有被当成可学习的数据。  

在根目录放原则性文档的解决方式对 AI 适用，但？？  

出于组织效率，个人的知识产出无法沉淀，个人优势不会被发现和复用。  
大量语料存在于 Drive，但未区分内容类型（建议是否被采纳？是否成功？）。  
让高效的局部行为，能引用到对应的“意图”。

## 核心模块

### 核心模块 1：Consensus History（共识历史）

功能：记录 Decision Log，而不仅是 Git Log。  
表现：用户修改 / 删除内容时，系统捕捉修改理由。  
价值：解决决策失忆。  

### 核心模块 2：What is Good（原则提取）

功能：从 Decision Log 中提取校准点，形成团队原则库。  
表现：自动提示是否将反复出现的判断设为系统约束。  
价值：一次性纠错 → 永久性知识。  

### 核心模块 3：Traceability（追溯验证）

功能：意图、实现、结果强绑定。  
表现：结果异常时，指回最初意图。  
价值：解决验证鸿沟。

## MVP

### 北极星版本

让用户感受到：  
“我在这个组织内，不用再重复教 AI 做事了”。  

实现闭环：  
意图确定 → 捕捉共识和知识 → 资产复用  

### 正在开发的 MVP（0.1）

筛选标准：  
•不依赖他人配合  
•不要求用户改变现有流程  
•能真实生成【可打标签的决策】记录
